### Token distribution on Sepolia

- KTM are minted to the deployer wallet at deployment.
- That deployer private key is imported into MetaMask as the "master wallet" on the Sepolia network.
- KTM are then manually sent from the master wallet to 3–5 test wallets via MetaMask's "Send" UI.
- Each test wallet self‑delegates via the UI upon connection.

Current architecture
Governor contract = source of truth for proposal creation and lifecycle.

API metadata (/api/proposals) = source of truth for app-side proposal details like category and display metadata.

proposalId = canonical join key between onchain proposal state and app metadata.

Magic embedded wallets = used for signing, no wallet-connect flow.

Important implemented behavior
If the account is still being prepared by the app/session layer, the UI shows setup messaging.

If tokens are present but self-delegation is missing, the UI shows Enable proposal.

Clicking Enable proposal triggers self-delegation through Magic signing.

After delegation, the UI refetches reads and unlocks Review proposal when the chain confirms eligibility.

Delegation failures show Something went wrong.

Full underlying errors are logged to the console.
